Methodology

This test progressively increases the VM count against a single storage target to find the density ceiling at a chosen per-VM offered load. Each VM runs an identical, rate-capped fio workload.

Phases

  • Phase 1 (doubling): 1 → 2 → 4 → 8 → ... VMs until write-p99 breaches the SLA.
  • Phase 2 (backfill): linear steps between the last passing and first failing count to pin the ceiling.

Per-step methodology fixes

  • Prefill: every VM sequentially writes the full test file before measurement, so the first-write allocation cost on thin-provisioned vSAN ESA / NFS VMDKs is not counted.
  • Sync barrier: all VMs in a step wait on a shared wall-clock epoch before starting fio, eliminating first-mover penalty where a single VM measures alone for tens of seconds.

Workload

  • mixed-70-30-rated.fio — 70/30 random read/write, 4k blocks, QD32, numjobs=1, rate-capped to keep offered load uniform across the ramp.
